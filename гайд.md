# Гайд: Каскадный кластер VPN Prime (NL + Москва)

## Схема итоговой архитектуры

```
Клиент (Россия)
    │
    │  VLESS Reality (выглядит как HTTPS к max.ru)
    ▼
┌─────────────────────────────┐
│   Московский сервер         │  ← entry node, клиент подключается сюда
│   Marzban + Xray            │
│   relay → NL                │
└────────────┬────────────────┘
             │  внутренний VLESS-тоннель (между серверами)
             ▼
┌─────────────────────────────┐
│   Нидерландский сервер      │  ← exit node, реальный выход в интернет
│   Marzban + Xray            │
│   vpn-panel (FastAPI)       │
│   Nginx                     │
└────────────┬────────────────┘
             │
             ▼
         Интернет
   (YouTube, Instagram, Telegram...)
```

## Часть 1 — Нидерландский сервер (если ещё не настроен)

### 1.1 Установка Marzban

```bash
# Подключиться к NL серверу
ssh root@NL_IP

# Установить Marzban одной командой
bash <(curl -sL https://github.com/Gozargah/Marzban/raw/master/script.sh) install

# Marzban установится в /opt/marzban
# Создать admin-аккаунт
marzban-cli admin create --sudo
# Ввести: имя пользователя → admin
# Ввести: пароль → придумать надёжный, записать
```

### 1.2 Настроить Reality inbound в Marzban (NL)

Открыть панель Marzban:
```
http://NL_IP:8100/dashboard
```

Войти → **Xray Settings (шестерёнка)** → вставить конфиг:

```json
{
  "log": {
    "loglevel": "warning"
  },
  "inbounds": [
    {
      "tag": "VLESS_TCP_REALITY",
      "listen": "0.0.0.0",
      "port": 443,
      "protocol": "vless",
      "settings": {
        "clients": [],
        "decryption": "none"
      },
      "streamSettings": {
        "network": "tcp",
        "security": "reality",
        "realitySettings": {
          "show": false,
          "dest": "max.ru:443",
          "xver": 0,
          "serverNames": ["max.ru", "www.max.ru"],
          "privateKey": "СГЕНЕРИРОВАТЬ_НИЖЕ",
          "shortIds": ["СГЕНЕРИРОВАТЬ_НИЖЕ"]
        }
      },
      "sniffing": {
        "enabled": true,
        "destOverride": ["http", "tls", "quic"]
      }
    }
  ],
  "outbounds": [
    {
      "tag": "direct",
      "protocol": "freedom"
    },
    {
      "tag": "block",
      "protocol": "blackhole"
    }
  ],
  "routing": {
    "domainStrategy": "IPIfNonMatch",
    "rules": [
      {
        "type": "field",
        "outboundTag": "block",
        "ip": ["geoip:private"]
      }
    ]
  }
}
```

**Сгенерировать ключи Reality для NL:**

```bash
# На NL сервере
docker exec marzban-marzban-1 xray x25519

# Вывод будет примерно такой:
# Private key: ABC123...
# Public key:  XYZ789...

# Записать оба ключа — понадобятся для Москвы тоже
```

**Сгенерировать shortId:**

```bash
openssl rand -hex 8
# Пример вывода: a1b2c3d4e5f6a7b8
```

Вставить `privateKey` и `shortId` в конфиг выше. Сохранить.

### 1.3 Запомнить данные NL сервера

```
NL_IP        = ваш IP нидерландского сервера
NL_PORT      = 443
NL_PUBLIC_KEY = публичный ключ из xray x25519
NL_SHORT_ID  = shortId из openssl rand
NL_SNI       = max.ru
NL_MARZBAN_USER = admin
NL_MARZBAN_PASS = ваш пароль
```

## Часть 2 — Московский сервер

### 2.1 Установка Marzban на Москве

```bash
# Подключиться к московскому серверу
ssh root@MOSCOW_IP

# Обновить систему
apt update && apt upgrade -y

# Установить Marzban
bash <(curl -sL https://github.com/Gozargah/Marzban/raw/master/script.sh) install

# Создать admin-аккаунт
marzban-cli admin create --sudo
# admin / придумать пароль (может быть тот же что на NL)
```

### 2.2 Создать сервисного пользователя на NL для каскада

Этот пользователь нужен для внутреннего тоннеля Москва → NL.
Обычные клиенты его не видят.

```bash
# На NL сервере — создать relay-пользователя через API
curl -X POST http://127.0.0.1:8100/api/admin/token \
  -d "username=admin&password=ПАРОЛЬ_NL" \
  | python3 -m json.tool

# Скопировать access_token из ответа

TOKEN="вставить_токен_сюда"

curl -X POST http://127.0.0.1:8100/api/user \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "relay_moscow",
    "proxies": {"vless": {"flow": "xtls-rprx-vision"}},
    "expire": 0,
    "data_limit": 0
  }' | python3 -m json.tool

# Из ответа скопировать "id" (UUID) пользователя relay_moscow
# Это RELAY_UUID
```

**Заменить в конфиге:**

| Плейсхолдер | Что подставить |
|---|---|
| `NL_IP` | IP нидерландского сервера |
| `RELAY_UUID` | UUID пользователя `relay_moscow` |
| `NL_PUBLIC_KEY` | Публичный ключ с NL сервера |
| `NL_SHORT_ID` | shortId с NL сервера |
| `СГЕНЕРИРОВАТЬ_ДЛЯ_МОСКВЫ` | Новые ключи, генерировать ниже |

**Сгенерировать ключи Reality для Москвы:**

```bash
# На московском сервере
docker exec marzban-marzban-1 xray x25519
# Записать privateKey и publicKey московского сервера

openssl rand -hex 8
# Записать shortId московского сервера
```

> **Важно:** У Москвы и NL — разные пары ключей.  
> Московский `publicKey` понадобится при добавлении Москвы в vpn-panel.

Сохранить конфиг → Marzban перезапустит Xray автоматически.

### 2.4 Проверить каскад

```bash
# На московском сервере проверить что тоннель в NL работает
docker exec marzban-marzban-1 curl -x socks5://127.0.0.1:1080 https://api.ipify.org

# Должен вернуть NL IP, а не московский
# Если не работает — проверить логи:
docker logs marzban-marzban-1 --tail 50
```

### 2.5 Открыть порты на московском сервере

```bash
ufw allow 443/tcp
ufw allow 8100/tcp   # Marzban API (только если нужен внешний доступ)
ufw reload

# Проверить
ufw status
```

> **Безопасность:** Порт 8100 лучше не открывать наружу.  
> Доступ к Marzban API Москвы из vpn-panel настроить через SSH-туннель (см. ниже).

## Часть 3 — Безопасный доступ к API Москвы

Чтобы vpn-panel (на NL) мог обращаться к Marzban API Москвы, есть два варианта:

### Вариант A — SSH-туннель (рекомендуется)

На нидерландском сервере создать постоянный туннель:

```bash
# Установить autossh
apt install autossh -y

# Создать systemd сервис для туннеля
nano /etc/systemd/system/moscow-tunnel.service
```

```ini
[Unit]
Description=SSH tunnel to Moscow Marzban API
After=network.target

[Service]
ExecStart=/usr/bin/autossh -M 0 -N \
  -o "ServerAliveInterval=30" \
  -o "ServerAliveCountMax=3" \
  -o "StrictHostKeyChecking=no" \
  -L 8101:127.0.0.1:8100 \
  root@MOSCOW_IP
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable moscow-tunnel
systemctl start moscow-tunnel

# Проверить
curl http://127.0.0.1:8101/docs
# Должен открыться Swagger Marzban московского сервера
```

Теперь API Москвы доступен с NL-сервера по адресу `http://127.0.0.1:8101`.

### Вариант B — Открыть порт с IP-ограничением

```bash
# На московском сервере — разрешить 8100 только с NL IP
ufw allow from NL_IP to any port 8100 proto tcp
ufw reload
```

Тогда API URL будет `http://MOSCOW_IP:8100`.

## Часть 4 — Добавить московский сервер в vpn-panel

### 4.1 Открыть админку vpn-panel

```
https://vpn-prime.ru/admin
```

### 4.2 Серверы → Добавить сервер

Заполнить форму:

```
Название:     Россия — Москва
IP адрес:     MOSCOW_IP
Локация:      Москва, Россия
Приоритет:    1   (NL = 0, Москва = 1, в подписке NL будет первым)

API URL:      http://127.0.0.1:8101   (если SSH-туннель)
              ИЛИ
              http://MOSCOW_IP:8100   (если открытый порт)

API User:     admin
API Pass:     пароль московского Marzban

Reality SNI:  gosuslugi.ru   (тот что указали в московском конфиге)

Минимальный тариф:  Lite   (доступен всем клиентам)
```

Нажать **Добавить**.

### 4.3 Проверить доступность

**Серверы → Пинг** (кнопка рядом с московским сервером).  
Должен вернуть: `Сервер доступен`.

## Часть 5 — Проверка полной цепочки

### 5.1 Создать тестового клиента

**Клиенты → Новый клиент:**
```
Имя:    Тест
Тариф:  Lite
Срок:   1 месяц
```

### 5.2 Проверить подписку

Открыть карточку клиента → скопировать Subscription URL → открыть в браузере:

```
https://vpn-prime.ru/sub/ВАШ_ТОКЕН/json
```

В JSON должно быть **два outbound**:
- `proxy` — нидерландский сервер
- `proxy-1` — московский сервер

### 5.3 Проверить через Happ

1. Happ → Добавить подписку → вставить URL
2. Подключиться через **московский** сервер
3. Открыть `https://api.ipify.org` — должен показать **нидерландский IP**
4. Открыть `https://vk.com` — должен работать (идёт напрямую)
5. Открыть `https://youtube.com` — должен работать (через каскад)

## Итоговая схема трафика

```
Клиент → Москва → NL → Интернет
                         ↑
                     YouTube, Instagram, Telegram

Клиент → Москва → DIRECT
                   ↑
               ВКонтакте, Сбербанк, Госуслуги, Яндекс
```

### Routing rules (где что идёт):

| Трафик | Путь | Причина |
|---|---|---|
| YouTube, Instagram, Telegram | Москва → NL → Интернет | Заблокированы в РФ |
| ВКонтакте, Сбербанк, Яндекс | Москва → Интернет напрямую | Не нужен VPN, быстрее |
| Госуслуги, ФНС, MOS.RU | Москва → Интернет напрямую | Требуют российский IP |
| Всё остальное | Москва → NL → Интернет | По умолчанию через каскад |

## Troubleshooting

### Московский сервер недоступен в vpn-panel

```bash
# На NL сервере — проверить туннель
systemctl status moscow-tunnel
curl http://127.0.0.1:8101/docs

# Если туннель упал — перезапустить
systemctl restart moscow-tunnel
```

### Клиент подключается к Москве, но IP остаётся московским

```bash
# На московском сервере — проверить логи Xray
docker logs marzban-marzban-1 --tail 100 | grep -E "nl-exit|error"

# Проверить что NL_IP, RELAY_UUID, NL_PUBLIC_KEY, NL_SHORT_ID правильные
# Пересохранить конфиг в Marzban → Xray перезапустится
```

### Клиент не получает московский сервер в подписке

```bash
# На NL сервере — проверить что пользователь создан на обоих серверах
# Зайти в Marzban NL и Marzban Москвы
# Найти пользователя клиента — он должен быть на обоих

# Если нет — в vpn-panel пересоздать клиента
# ИЛИ запустить sync вручную через Дашборд → Синхронизировать трафик
```

### YouTube не открывается через московский сервер

```bash
# Проверить routing rules в московском Xray
# Убедиться что правило nl-exit стоит ПОСЛЕДНИМ в routing.rules
# Порядок важен: block → direct (ru) → nl-exit

# Проверить что RELAY_UUID совпадает с UUID пользователя relay_moscow на NL
marzban-cli user show relay_moscow  # на NL сервере
```
